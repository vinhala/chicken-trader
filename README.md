# AI-Powered Daily Investment Opportunity MVP

This repository contains an MVP implementation of a news-driven investment opportunity platform with:

- **Frontend**: Vue 3 + Vite + Pinia + Vue Router
- **Backend**: Python FastAPI + SQLAlchemy + Celery
- **Data**: PostgreSQL + Redis
- **Deployment target**: Docker Swarm stack with `lucaslorentz/caddy-docker-proxy:latest`

## Features implemented

- Email/password registration and login
- Mandatory disclaimer acceptance with timestamped consent storage
- Daily opportunities dashboard with event reports
- Opportunity detail with suggested securities
- Follow thesis workflow
- Daily reevaluation states: Active / Hold / Warning / Sell
- Terminal Sell behavior: single notification + close thesis + stop reprocessing
- In-app notifications center
- Watchlist management
- Broker settings and broker-aware asset filtering
- News ingestion worker and AI report generation pipeline with ticker validation retry

## Architecture

- `frontend/` Vue single-page app
- `backend/` FastAPI API, service layer, models, Celery tasks
- `docker-compose.network.yml` caddy + overlay networks
- `docker-compose.app.yml` core application services
- `docker-compose.urls-local.yml` caddy routing labels for local development
- `docker-compose.urls-prod.yml` caddy routing labels for production
- `ops/swarm/` deployment runbook

## Local development

### 1) Build local images

```bash
cp .env.example .env
make build-local
```

### 2) Deploy locally on Swarm

```bash
make deploy-local
```

### 3) Backend

```bash
cd backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 4) Frontend

```bash
cd frontend
npm install
npm run dev
```

Set `VITE_API_BASE_URL` if needed; default is `/api`.

`DATABASE_URL` and `REDIS_URL` are auto-generated by the backend from `POSTGRES_*` and `REDIS_*` variables unless you explicitly override them.

## Swarm deployment

See `ops/swarm/README.md`.

Quick start:

```bash
docker build -t chicken-trader-backend:latest ./backend
docker build -t chicken-trader-frontend:latest ./frontend
docker swarm init
docker stack deploy -c docker-compose.network.yml chicken-trader
docker stack deploy -c docker-compose.network.yml -c docker-compose.app.yml -c docker-compose.urls-local.yml chicken-trader
```

## API summary

- `POST /api/auth/register`
- `POST /api/auth/login`
- `GET /api/auth/me`
- `GET /api/opportunities`
- `GET /api/opportunities/{event_id}`
- `POST /api/opportunities/{event_id}/follow`
- `GET /api/theses`
- `POST /api/theses/reevaluate`
- `GET /api/notifications`
- `POST /api/notifications/{id}/read`
- `GET|POST|DELETE /api/watchlist`
- `GET|PUT /api/broker/settings`

## Background jobs

- `ingest_news_task`: fetch, dedupe, cluster, rank, generate reports
- `reevaluate_theses_task`: assign lifecycle state, enforce terminal Sell behavior
